Architecture & Data Flow

Two-phase dynamic CLI with Discovery Document-driven command surface

Data Flow Diagram

  User Input (CLI args: gws drive files list --params '{"pageSize":5}')
       |
       v
  [main.rs] Phase 1: Extract service name ("drive")
       |
       v
  [discovery.rs] Fetch/cache Discovery Document
       |  GET https://www.googleapis.com/discovery/v1/apis/drive/v3/rest
       |  Cache: ~/.config/gws/cache/ (24h TTL)
       v
  [commands.rs] Build dynamic clap CLI tree from Discovery
       |  resources -> subcommands, methods -> leaf commands
       v
  [main.rs] Phase 2: Parse remaining args through clap
       |
       v
  [helpers/mod.rs] Check for service-specific helper
       |  (e.g., gmail +send, drive +upload)
       v
  [auth.rs] Authenticate (priority chain):
       |  1. GOOGLE_WORKSPACE_CLI_TOKEN env var
       |  2. GOOGLE_WORKSPACE_CLI_CREDENTIALS_FILE
       |  3. ~/.config/gws/credentials.enc (AES-256-GCM)
       |  4. ~/.config/gws/credentials.json
       |  5. GOOGLE_APPLICATION_CREDENTIALS (ADC)
       |  6. ~/.config/gcloud/application_default_credentials.json
       v
  [executor.rs] Build & send HTTP request:
       |  URL: {baseUrl}/{servicePath}/{method.path}
       |  Headers: Authorization: Bearer {token}
       |           x-goog-api-client: gl-rust/gws-0.7.0
       |  Body: --json payload (if applicable)
       |  Upload: --upload file (multipart)
       v
  [client.rs] Retry logic (429 rate limit, Retry-After, 3 max retries)
       |
       v
  [executor.rs] Process response:
       |  Parse JSON/binary
       |  Optional: Model Armor sanitization
       |  Pagination: --page-all / --page-limit / --page-delay
       v
  [formatter.rs] Format output (--format json|table|yaml|csv)
       |
       v
  stdout

Module Structure

main.rs

~700 lines - Entry point

Two-phase argument parsing, service routing, method resolution. Routes to auth, schema, MCP, or API execution.

executor.rs

~2000 lines - HTTP execution

Request construction, multipart uploads, pagination, response formatting, Model Armor sanitization, dry-run mode.

setup.rs

~2040 lines - Setup wizard

OAuth setup flow: loopback server, gcloud integration, scope prompts, project selection, API enablement.

setup_tui.rs

~1738 lines - Terminal UI

Interactive TUI for scope selection, service picker, checkbox lists using ratatui + crossterm.

mcp_server.rs

~1151 lines - MCP server

Model Context Protocol stdio JSON-RPC server. Exposes Google APIs as tools for Claude/LLM integration.

auth.rs

~642 lines - Authentication

OAuth2 and Service Account token retrieval. Credential loading priority chain with encrypted token caching.

validate.rs

~569 lines - Input validation

Path traversal prevention, control char rejection, API identifier validation, symlink safety.

schema.rs

~449 lines - Schema introspection

gws schema command for inspecting API method schemas, parameters, and request/response types.

credential_store.rs

~341 lines - Encryption

AES-256-GCM encryption/decryption of credential files. OS keyring integration with file fallback.

commands.rs

~276 lines - CLI builder

Recursive clap::Command tree builder from Discovery Document resources and methods.

discovery.rs

~300 lines - Discovery

Serde models for Discovery Document. Fetch from Google, cache locally with 24h TTL.

services.rs

~178 lines - Service registry

Maps service aliases (drive, gmail, sheets...) to Discovery API name/version.

helpers/*

~3500 lines - Service helpers

12 service-specific helpers: Gmail (send, watch, triage), Drive, Sheets, Docs, Chat, Calendar, Script, Events, Model Armor, Workflows.

formatter.rs

~200 lines - Output formatting

JSON (pretty), Table (aligned columns), YAML, CSV output formatters.

Local Data Storage Map

~/.config/gws/                           (or $GOOGLE_WORKSPACE_CLI_CONFIG_DIR)
  |-- credentials.enc                    AES-256-GCM encrypted OAuth credentials
  |-- credentials.json                   Plaintext OAuth credentials (legacy)
  |-- token_cache.json                   Encrypted access token cache (scope-keyed)
  |-- sa_token_cache.json                Service Account token cache
  |-- .encryption_key                    Base64 256-bit key (fallback, 0600 perms)
  |-- client_secret.json                 OAuth client config
  |-- cache/
       |-- drive_v3.json                 Cached Discovery Documents (24h TTL)
       |-- gmail_v1.json
       |-- ...
