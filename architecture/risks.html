Risks & Known Issues

Open issues, identified risks, and harm assessment

Risk Cards

Unsafe File Path Handling in --upload / --output MEDIUM

GitHub Issue #251 reports that --upload and --output flags may accept unsafe filesystem paths. While validate.rs provides path validation helpers, they may not be applied to all code paths.

Issue: #251 "Security: dynamic --upload / --output accept unsafe filesystem paths"
Status: OPEN
Mitigation: validate.rs provides validate_safe_output_dir() and validate_safe_dir_path()
Discovery Document Fetch Failures (HTTP 404) MEDIUM

Issues #255 and #256 report Discovery Document fetch failures on macOS with rustls-tls, causing all API commands to fail. This is a reliability issue, not a security issue, but affects all functionality.

Issue: #255 "All API commands fail: Discovery Document fetch returns 404 (macOS, v0.6.3)"
Issue: #256 "Discovery Document fetch fails with rustls-tls on macOS (HTTP 404)"
Status: OPEN
Non-ASCII Subject Encoding in Gmail LOW

Issue #263 reports that gws gmail +send incorrectly encodes non-ASCII subject lines, which could lead to garbled email subjects. Information loss risk for internationalized email.

Issue: #263 "gws gmail +send incorrectly encodes non-ASCII subject lines"
Status: OPEN
OAuth Scope Mismatch in Gmail Triage LOW

Issue #265: gmail +triage selects metadata scope but fails with 403 when using query parameter. Functional issue causing command failure.

Issue: #265 "gmail +triage selects metadata scope and fails with q parameter (403)"
Status: OPEN
Nested Resource Commands Fail LOW

Issue #262: Nested resource commands produce "unrecognized subcommand" errors. This is a functional bug in the dynamic command tree builder.

Issue: #262 "Nested resource commands fail with 'unrecognized subcommand'"
Status: OPEN
UTC Timezone Assumption in Calendar LOW

Issue #259: agenda --today uses UTC for day boundaries, producing wrong results in non-UTC timezones. Information loss risk for calendar queries.

Issue: #259 "agenda --today uses UTC for day boundaries, wrong results in non-UTC timezones"
Status: OPEN

Recommendations

  1. Address Issue #251 (file path safety) - Ensure all --upload and --output code paths use the existing validate.rs helpers.
  2. Fix Discovery fetch failures (#255, #256) - Investigate rustls-tls compatibility on macOS. Consider fallback to native-tls feature.
  3. Fix non-ASCII encoding (#263) - Implement RFC 2047 encoding for email subject lines.
  4. Fix timezone handling (#259) - Use local timezone for day boundaries in calendar commands.
  5. Consider request timeouts - Add configurable HTTP request timeouts (currently relies on reqwest defaults).
